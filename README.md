# 基於C語言之多使用者支援與權限控管之虛擬檔案系統技術開發

### 技術開發背景及目標規劃

- 技術開發動機及目標

  現今資訊隱私性日漸重要，造就如Edge AI之興起，而最計算機基礎建設較少Personal Privacy技術，本組透過C語言開發一套Unix like的檔案管理系統，模仿了C語言的架構及特性，例如inode,  VFS架構...etc, 希冀達到良好的權限及隱私管理

- 開發規劃
  
  透過「.c」、「.h」進行檔案結構上的管理，使用Makefile管理整體環境
  根據「實際用途」以及「資料結構」不同進行檔案區分


- 開發規劃
  
  系統檔案結構的劃分主要依據「實際用途」與「資料結構層級」進行設計，例如將路徑解析、目錄操作、檔案存取、空間管理與權限控制分別實作於不同模組中。
  此種分工方式不僅有助於系統擴充，也能清楚呈現各層模組在檔案系統中的角色與責任。
  透過此開發方式，本專題不僅完成一套可運作的檔案管理系統，更藉由實際程式實作與實驗操作，驗證檔案系統在權限管理與資料隱私保護上的基本設計原則，作為未來進一步延伸至多使用者、加密存儲或進階存取控制機制的基礎。

### 實驗及結果

- 實驗
  
  請參考影片

- 結果
  
  本技術實作並驗證了一套簡化之虛擬檔案系統（Virtual File System, VFS），系統整合了路徑解析、目錄結構管理、inode 檔案描述、block bitmap 空間配置以及權限控制等核心模組，並透過多項指令操作進行功能驗證。
  系統不僅能完成基本檔案操作，亦能在錯誤情境下維持穩定與一致性，顯示其整體架構具備良好的可行性與延展性。

- 討論
  
  本系統整合了路徑解析、dentry tree、inode 與 block bitmap 等核心模組，並於所有檔案操作前執行權限檢查，實現基本的存取控制機制。透過 inode 權限位元，系統能有效限制未授權的讀取、寫入與目錄操作，符合傳統 UNIX 檔案系統的安全設計原則。
  此外，所有操作皆需先完成路徑解析並定位至合法 dentry，避免未經驗證的路徑直接存取底層結構，有助於降低錯誤輸入或惡意操作對系統造成的影響。在資料存儲層，block bitmap 集中管理資料區塊配置，確保檔案內容與實體空間的一致性，並避免不同檔案共用相同區塊。

- 限制
  
  本系統未實作使用者登入與密碼驗證機制，僅假設單一使用者身分，無法防範未經授權的系統存取，權限模型亦僅支援基本的讀、寫、執行位元，未提供更細緻的存取控制機制，如 ACL 或能力導向安全模型。此外，系統設計假設單執行緒環境，未考慮並行存取與競爭條件，可能在多使用者情境下影響 inode 與 block bitmap 的一致性。目前所有資料皆以明文方式儲存，亦未提供加密、稽核或安全刪除等進階資安機制，仍有進一步強化的空間。
